# retro

> I regularly use this skill with great success.

Runs a guided end-of-session retrospective. It reviews the session evidence, discusses one
observation at a time, records confirmed follow-ups, and writes a durable retrospective note.

## Workflow

1. Read the session transcript and relevant subagent transcripts.
2. Build a visible agenda from evidence found in the session.
3. Discuss one item at a time.
4. Record confirmed follow-ups.
5. Depending on the backend, gives the option of automatically writing and verifying the
   retrospective note in a file instead of spending more time, attention, and tokens on it in the
   current session.

Substantive follow-up work normally belongs in a fresh session. Small corrections can be made
during the retrospective with the user's approval.
