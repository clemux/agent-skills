# plan-mode-tdd

> I was using this at some point but I don't think it makes sense with current
> harnesses and models. When it does make sense,
> [superpowers](https://github.com/obra/superpowers) is better anyway.

The skill asks an agent to split work into behavior changes and repeat:

1. Write or update a failing test.
2. Add the minimal implementation.
3. Run the tests.
4. Run `feature-dev:code-reviewer` and address its findings.
5. Commit.
